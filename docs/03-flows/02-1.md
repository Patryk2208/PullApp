## Actors: Passenger
## Preconditions: Passenger logged in

### All acceptable routes are returned ranked to the passengers, passengers choose one, it's then sent to the Driver who can accept or Denyss

## Flow:
1. Passenger interface → API Gateway → Trip Planner
   POST /api/passenger/route-request
   {start, end, ...(additional requirements)}

2. Trip Planner validates:
   - In service area (PostGIS query)
   - ...

3. Trip Planner pushes event to ComputeQueue "match route" and stores id to cache

4. Route-Calc computes most fitting routes and ranks them

5. Route-Calc pushes results to ResultsQueue

6. Trip Planner reads from Results Queue

7. Trip Planner → API Gateway → Driver app
   Response: { { route_request_id, [ { route_id, eta_to_destination } ] }

8. Passenger chooses Route | Passenger interface → API Gateway → Trip Planner
    POST /api/passenger/route-request/{route_request_id}
    { route_id }

9. Trip Planner pushes event "Route requested" to Event Queue
    { route_request_id, passenger_info, additional_info }

10. Notification Service reads from Event Queue and sends a notification to Driver

11. Driver accepts or not
    POST /api/driver/confirmation/{route_request_id}
    { yes/no }

IF YES:
1. Driver accepts, Trip Planner emits "Route Accepted"
2. Notification Service sends notification to Passenger
3. Passenger Interface is moved to a [Ride Screen] with a Chat from Chat Service for them to set up the details 

IF NO:
1. Driver declines, Trip Planner emits "Route Declined"
2. Notification Service sends notification to Passenger
3. Passenger Interface shows info and goes back to the [Route Choice Screen]

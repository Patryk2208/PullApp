C4 model approach, architecture described in 4 layers:
1. Context diagrams, so a rough image of our business logic and main flows, we can say it's already done in the docs
2. Container diagrams, a general structure of the app, the main services that should be created and interactions, events and storage, this should also correspond with context boundaries and the division of each use-case scope
3. Component diagrams,implementation and linking appropriate use-cases, also how sub-components interact with other components defined in use-cases
4. Code Diagrams, a specified architecture of each module, app design, tech choices and internal code structure
